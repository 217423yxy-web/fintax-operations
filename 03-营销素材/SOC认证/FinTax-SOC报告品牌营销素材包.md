### 页面正文

#### 引导语

FinTax 视客户数据安全为最高优先级。我们的系统已通过独立第三方审计机构的严格审查，获得 SOC 1 Type 2 和 SOC 2 Type 2 双重认证，为全球客户提供经过验证的安全保障。

#### SOC 2 Type 2 板块

**SOC 2 Type 2 — 安全、可用性与保密性认证**

FinTax 的专业税务与财务管理软件系统已通过 SOC 2 Type 2 审计，覆盖安全性（Security）、可用性（Availability）和保密性（Confidentiality）三大信任服务类别。该审计由 A-LIGN Assurance 依据 AICPA 2017 信任服务准则执行，确认 FinTax 在审计期间的控制设计合理且运行有效。

**这意味着什么：** 您的加密资产财务数据在 FinTax 平台上受到企业级安全控制的保护，包括数据加密（传输中和静态）、基于角色的访问控制、入侵检测系统、定期漏洞扫描和渗透测试等多层防护。

#### SOC 1 Type 2 板块

**SOC 1 Type 2 — 财务报告内控认证**

FinTax 同时持有 SOC 1 Type 2 报告，该报告依据 SSAE 18 标准，专门评估与用户实体财务报告相关的内部控制。审计结果确认 FinTax 的 IT 通用控制在审计期间设计适当且运行有效。

**这意味着什么：** 当您使用 FinTax 处理加密资产的财务会计和税务申报时，相关控制环节已经过独立验证，您的审计师可以直接依赖这份报告来评估与 FinTax 服务相关的控制风险。

#### 审计覆盖的关键领域

- 控制环境与治理
- 风险评估与管理
- 变更控制与系统开发生命周期
- 计算机运营（可用性与备份）
- 数据通信与网络安全
- 信息安全与访问管理
- 业务连续性与灾难恢复

#### 持续合规承诺

FinTax 将合规视为持续性工程，而非一次性项目。我们的 SOC 审计覆盖了连续的运行期间，且我们承诺每年持续接受独立审计。如需获取报告副本，请联系 support@fintax.tech。

### 推文 2：科普向——什么是 SOC 报告

"SOC 报告到底是什么？"

简单说，它是由独立审计师对一家公司的安全和内控体系进行深度检查后出具的"体检报告"。

SOC 2 关注安全性、可用性、保密性
SOC 1 关注财务报告相关的内控

Type 2 意味着不只看设计，还要验证一段时间内的实际运行。

门槛很高，这也是为什么加密行业持有 SOC 报告的公司屈指可数。

### 帖子 2：思想领导力帖

**标题：为什么加密资产行业需要更多 SOC 审计？**

最近我一直在思考一个问题：为什么在传统金融服务领域早已成为标配的 SOC 审计，在加密资产行业却依然稀缺？

SOC（Service Organization Control）报告是 AICPA 体系下的权威认证。它不是自我声明，而是由独立审计机构对企业的内控体系进行系统性检验后出具的专业报告。Type 2 审计更是要求在一段时间内持续验证控制的运行有效性，而非仅仅评估设计。

对于处理客户财务数据的服务商来说，这类审计应该是最低标准，而不是锦上添花。

FinTax 在今年同时完成了 SOC 1 Type 2 和 SOC 2 Type 2 审计，覆盖了安全、可用、保密和财务报告内控等维度。我们希望通过这些实际行动，推动整个行业对合规标准的重视。

行业的信任不是一家公司能建立的，但每一家选择主动接受审查的公司，都在为行业的成熟做出贡献。
